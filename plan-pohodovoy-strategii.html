1. Концепция и Обзор

Название игры

«Век Империй»

Пошаговая стратегия в фэнтезийном сеттинге. Игрок развивает своё королевство, управляет армиями, экономикой и дипломатией на большой карте.

ЖАНР

Пошаговая 4X / Гранд-стратегия (лёгкая)

ПЛАТФОРМА

Браузер (HTML + JS + Canvas / DOM)

РЕЖИМЫ

Одиночная vs AI • Локальный hot-seat • WebRTC мультиплеер (позже)

2. Основные Механики

Карта и Движение

  • • Квадратная или гекс-карта (рекомендую hex для тактики)
  • • Тайлы: равнина, лес, гора, река, вода
  • • Затраты движения (AP) в зависимости от типа юнита и местности
  • • Видимость (fog of war)

Экономика и Ресурсы

  • • Золото, Еда, Дерево, Камень, Железо (или мана)
  • • Города производят ресурсы + очки науки/культуры
  • • Содержание армии стоит золото каждый ход
  • • Торговля и дипломатия (позже)

Бой и Развитие

  • • Пошаговый бой на карте или отдельный экран
  • • Опыт, уровни, прокачка юнитов
  • • Герои с навыками и артефактами
  • • Осада городов, захват территорий

3. Структура Хода (Turn Phases)

  1. 1. Фаза Экономики — сбор ресурсов, содержание армии, производство в городах
  2. 2. Фаза Движения — перемещение юнитов, разведка
  3. 3. Фаза Действий — атака, строительство, использование способностей
  4. 4. Фаза Боя — разрешение всех боёв (можно в реальном времени или пошагово)
  5. 5. Конец Хода — проверка победы/поражения, события, AI ход

4. Архитектура Классов (OOP)

Игра построена на чистых JavaScript классах (ES6+). Каждый игровой объект — экземпляр класса. Это позволит легко расширять механики, добавлять новые типы юнитов/зданий и делать сохранения.

Пример реализации (JS)

class Unit {
  constructor(type, owner, x, y) {
    this.id = Date.now() + Math.random();
    this.type = type;           // 'warrior', 'archer'...
    this.owner = owner;
    this.x = x; this.y = y;
    this.hp = 100;
    this.maxHp = 100;
    this.attack = 15;
    this.defense = 8;
    this.movement = 3;
    this.level = 1;
    this.experience = 0;
  }

  move(toX, toY, gameMap) {
    const dist = Math.abs(this.x - toX) + Math.abs(this.y - toY);
    if (dist > this.movement) return false;
    // проверка местности...
    this.x = toX; this.y = toY;
    return true;
  }

  attack(target) {
    if (!target || target.owner === this.owner) return;
    const damage = Math.max(1, this.attack - target.defense);
    target.takeDamage(damage);
    this.experience += 10;
    if (this.experience >= this.level * 50) this.levelUp();
  }

  takeDamage(dmg) {
    this.hp = Math.max(0, this.hp - dmg);
    if (this.hp <= 0) this.die();
  }

  levelUp() {
    this.level++;
    this.maxHp += 15;
    this.hp = this.maxHp;
    this.attack += 3;
    this.defense += 2;
  }

  die() {
    // удалить из игры
    console.log(`${this.type} погиб`);
  }
}
Ключевые принципы
  • • Композиция > наследование (Unit может иметь разные компоненты: Movement, Combat, Economy)
  • • Система событий (EventEmitter) для loose coupling
  • • Сериализация всех объектов для сохранения (JSON)
  • • Разделение логики и отрисовки (Renderer отдельно)

5. План Разработки (MVP → Полная версия)

MVP

Базовая карта + 2-3 типа юнитов + 1 город + простой AI

Ход игрока → ход AI → победа при захвате столицы

v1

Полная экономика, строительство зданий, опыт юнитов, туман войны

v2

Герои, артефакты, дерево технологий, события, лучшая графика (Canvas)

v3

WebRTC мультиплеер, редактор карт, кампания, сохранения в облаке
