1. Цели и концепция

Создать браузерную пошаговую стратегию в фэнтезийном сеттинге с акцентом на глубину механик, удобную архитектуру кода и возможность дальнейшего расширения (включая мультиплеер).

  • Жанр: Пошаговая стратегия / 4X-лайт
  • Платформа: Браузер (чистый HTML + JS + Canvas/DOM)
  • Режимы: Одиночная игра против AI, hot-seat, WebRTC мультиплеер (в перспективе)

2. Основные механики

Карта и перемещение

  • Квадратная или гексагональная карта (рекомендуется hex)
  • Типы тайлов: равнина, лес, гора, река, вода, болото
  • Очки действия (AP) для перемещения в зависимости от типа юнита и местности
  • Туман войны

Экономика и ресурсы

  • Золото, Еда, Дерево, Камень, Железо (или Мана)
  • Города производят ресурсы + очки науки/культуры
  • Содержание армии стоит золото каждый ход

Бой и развитие

  • Пошаговый или полу-реал-тайм бой на карте
  • Система опыта и уровней юнитов
  • Герои с навыками и артефактами (позже)
  • Осада и захват городов

3. Структура хода (фазы)

  1. Фаза экономики — сбор ресурсов, содержание армии, производство в городах
  2. Фаза движения — перемещение юнитов, разведка
  3. Фаза действий — атака, строительство, способности
  4. Разрешение боёв
  5. Конец хода — события, проверка условий победы, ход AI

4. Архитектура классов (JavaScript)

Игра строится на чистых ES6-классах. Это позволит легко расширять механики, добавлять новые типы юнитов/зданий и делать сохранения.

Пример реализации класса Unit

class Unit {
  constructor(type, owner, x, y) {
    this.id = Date.now() + Math.random();
    this.type = type;
    this.owner = owner;
    this.x = x; this.y = y;
    this.hp = 100; this.maxHp = 100;
    this.attack = 15; this.defense = 8;
    this.movement = 3;
    this.level = 1; this.experience = 0;
  }

  move(toX, toY, gameMap) {
    const dist = Math.abs(this.x - toX) + Math.abs(this.y - toY);
    if (dist > this.movement) return false;
    this.x = toX; this.y = toY;
    return true;
  }

  attack(target) {
    if (!target || target.owner === this.owner) return;
    const damage = Math.max(1, this.attack - target.defense);
    target.takeDamage(damage);
    this.experience += 10;
    if (this.experience >= this.level * 50) this.levelUp();
  }

  takeDamage(dmg) {
    this.hp = Math.max(0, this.hp - dmg);
    if (this.hp <= 0) this.die();
  }

  levelUp() {
    this.level++;
    this.maxHp += 15;
    this.hp = this.maxHp;
    this.attack += 3;
    this.defense += 2;
  }

  die() { console.log(`${this.type} погиб`); }
}

5. План разработки (поэтапно)

MVP
Базовая карта + 2–3 типа юнитов + 1 город + простой AI. Ход игрока → ход AI → победа при захвате столицы.
v1.0
Полная экономика, строительство зданий, опыт юнитов, туман войны, несколько городов.
v2.0
Герои, артефакты, дерево технологий, события, улучшенная графика (Canvas).
v3.0
WebRTC мультиплеер, редактор карт, кампания, облачные сохранения.
Документ готов для согласования. При необходимости могу сразу приступить к реализации MVP на основе этого плана.
